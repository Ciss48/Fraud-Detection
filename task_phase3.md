The problem to solve Phase 2 left us with a precision-recall tradeoff that no single method solved well: - Simple threshold (amt > $500): decent F1 (29.9%) but only catches half the fraud - Association Rules: catch 85.9% of fraud but trigger on 510K transactions — unusable standalone - Isolation Forest: worst of both worlds (F1=22.4%) Phase 3 trains ML models that learn which combinations of signals matter, rather than hardcoding thresholds or rules. --- Step by step 3.1 Load Data Loads transactions_phase2.csv which already has flag_threshold, flag_iso_forest, and flag_rules columns from Phase 2. 3.2 Feature Engineering Builds two feature sets: - Base (9 features): amt, hour_of_day, day_of_week, month, distance_km, city_pop, age, is_night, category_enc - Rule-augmented (11 features): base + flag_threshold + flag_rules 3.3 Cross-Validation Setup Uses 5-fold Stratified CV with cross_val_predict to get out-of-fold probabilities — this gives honest evaluation on all 1.85M rows without data leakage. Also defines a shared evaluate() helper that prints Precision/Recall/F1/AUC-PR for any model. 3.4 Random Forest (Base) - class_weight='balanced' to handle the 1:190 imbalance (no SMOTE needed — confirmed in EDA) - n_estimators=300, max_depth=12 - After getting OOF probabilities, sweeps thresholds 0.1→0.9 to find the decision cutoff with the best F1 (default 0.5 is rarely optimal for imbalanced data) 3.5 XGBoost (Base) - Uses scale_pos_weight=190 (XGBoost's equivalent of balanced class weighting) - eval_metric='aucpr' to optimize for the right metric during training - Same threshold sweep as RF 3.6 Rule-Augmented RF Identical RF config but trained on the 11-feature set that includes flag_threshold and flag_rules. The model can learn: "when the rules fire AND amt is high, this is almost certainly fraud" — something neither rules alone nor ML alone can express. 3.7 Precision-Recall Curves Overlays all 3 ML model curves on one chart, with Phase 2 baselines plotted as scatter points. AUC-PR is the headline metric (better than AUC-ROC for imbalanced data). 3.8 Feature Importance Fits both RF models on the full dataset (not CV) to extract feature importances. Side-by-side comparison shows how much weight the rule flags carry in the augmented model vs the raw engineered features. 3.9 Full Comparison Chart 6-method Precision/Recall/F1 bar chart covering everything from Phase 2 and Phase 3 together — the complete story of the project. 3.10 Confusion Matrix Auto-selects the best model by F1, shows the TP/FP/FN/TN breakdown so you can see concretely: how many fraud cases were caught, how many false alarms were raised. 3.11 Save Outputs - outputs/rules/phase3_method_comparison.csv — final metrics table - data/processed/transactions_phase3.csv — full dataset with model probabilities and predictions for all 3 models
